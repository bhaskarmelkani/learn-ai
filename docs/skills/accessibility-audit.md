# Accessibility Audit

Use this workflow when a change affects semantics, keyboard interaction, motion, color contrast, or screen-reader clarity.

## Check For

- semantic headings and landmarks
- keyboard navigation and visible focus states
- contrast that stays readable in light and dark themes
- motion that does not block comprehension
- lab controls and status messaging that remain understandable without visual guesswork

## Goal

Keep the learning experience accessible as new chapters, labs, and UI patterns are added.
