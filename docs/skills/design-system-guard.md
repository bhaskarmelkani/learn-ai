# Design System Guard

Use this workflow when a change could drift away from the repo's established UI language.

## Check For

- reuse of existing components before new one-off patterns
- spacing, radii, and typography that match adjacent screens
- mobile and dark-mode consistency
- accessible focus states and button semantics
- shared treatment for browser-AI badges, status panels, and control layouts

## Goal

Keep the product visually coherent so new lessons and labs feel like part of the same system.
