# From Text to Tokens, Embeddings, Attention, and LLMs

By now we have the big pattern:

- models map inputs to outputs
- parameters are learned from data
- neural networks learn useful internal representations

The final step is to ask: how do we apply that same story to language?

## Step 1: Break Text into Tokens

A language model does not receive "meaning" directly.

It first receives a sequence of **tokens**.

A token might be:

- a whole word
- part of a word
- punctuation
- whitespace or formatting

## Step 2: Turn Tokens into Vectors

Neural networks operate on numbers, not raw text.

So each token is turned into a vector of numbers called an **embedding**.

You can think of an embedding as the model's learned numeric representation of a token.

| Token | Idea |
|-------|------|
| `dog` | one learned vector |
| `cat` | another learned vector |
| `(` | another learned vector |
| `ing` | another learned vector |

## Step 3: Let Tokens Influence Each Other

For language, context matters.

The token after "The cat sat on the" should be influenced by what came before it.

That need leads to **attention**: a way for tokens to weigh which other tokens matter for the next prediction.

## Step 4: Predict the Next Token

Once the model has token embeddings and context-aware representations, it produces a probability distribution over possible next tokens.

## The Architecture Name

The architecture built around attention is called the **transformer**.

When we train a transformer at large scale on next-token prediction, then add instruction-following and preference tuning, we get what people call an **LLM**.

## The Smooth Connection

So the progression is not:

"simple models ... and then somehow magic LLMs."

It is:

1. models learn numbers
2. lines learn simple patterns
3. non-linear models learn curves
4. neural networks learn internal features
5. transformers let language tokens use context
6. scale turns that into a powerful language model

<Checkpoint prompt="What is the role of embeddings?">
Embeddings turn discrete tokens into learned numeric vectors so a neural network can work with them.
</Checkpoint>

<Callout type="idea" title="Takeaway">
An LLM is still the same core story: a model with learned parameters, trained to make better predictions. The main differences are language input, attention-based architecture, and scale.
</Callout>
