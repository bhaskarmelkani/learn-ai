# Neural Networks

Here's the key insight: **a neural network is just many small models connected together**.

Each neuron computes a weighted sum and then applies an activation:

$$z = w_1 x_1 + w_2 x_2 + \cdots + w_n x_n + b$$
$$a = \sigma(z)$$

Sound familiar? A **sigmoid neuron** looks exactly like logistic regression. In modern networks, hidden layers often use activations like ReLU instead of sigmoid, but the core pattern is the same: linear transform, then non-linearity.

<Callout type="idea">
A neural network is not magic. It's layers of simple functions (linear transform + non-linearity) composed together. Each layer learns to extract increasingly abstract features.
</Callout>

## From One Neuron to a Layer

A **layer** is just multiple neurons processing the same input in parallel:

```
Input (3 features)          Layer (4 neurons)         Output
     x₁ ─────────────────→  [n₁]  → a₁
     x₂ ─────────────────→  [n₂]  → a₂
     x₃ ─────────────────→  [n₃]  → a₃
                             [n₄]  → a₄
```

Each neuron has its own weights and bias. Each learns to detect a different pattern.

In matrix form, a layer is:

$$\mathbf{a} = \sigma(\mathbf{W}\mathbf{x} + \mathbf{b})$$

Where $\mathbf{W}$ is a matrix of all the weights.

## Stacking Layers: The Deep in Deep Learning

```
Input      Hidden Layer 1    Hidden Layer 2    Output
 [x₁]        [n₁]              [n₅]
 [x₂] ──→    [n₂]    ──→       [n₆]    ──→   [ŷ]
 [x₃]        [n₃]              [n₇]
              [n₄]
```

Each layer transforms the data, making it easier for the next layer:

| Layer | What it learns (image example) |
|-------|-------------------------------|
| Layer 1 | Edges, gradients |
| Layer 2 | Textures, corners |
| Layer 3 | Parts (eyes, wheels) |
| Layer 4 | Objects (faces, cars) |

## Why Non-Linearity Matters

Without the activation function $\sigma$, stacking layers is pointless:

$$\mathbf{W}_2(\mathbf{W}_1 \mathbf{x} + \mathbf{b}_1) + \mathbf{b}_2 = \mathbf{W}'\mathbf{x} + \mathbf{b}'$$

Two linear layers = one linear layer. The non-linearity is what gives depth its power.

<Checkpoint prompt="Why can't we just stack many linear layers and call it deep learning?">
Because the composition of linear functions is still just another linear function. Without a non-linearity, depth adds complexity in implementation but not in expressive power.
</Checkpoint>

## Common Activation Functions

| Function | Formula | Used For |
|----------|---------|----------|
| Sigmoid | $\frac{1}{1+e^{-z}}$ | Output (binary classification) |
| ReLU | $\max(0, z)$ | Hidden layers (most common) |
| Softmax | $\frac{e^{z_k}}{\sum e^{z_j}}$ | Output (multi-class) |

**ReLU** (Rectified Linear Unit) is the workhorse of modern neural networks — dead simple, fast to compute, and works great in practice.

## Architecture = Choosing the Shape

Designing a neural network means choosing:

1. **How many layers?** (depth)
2. **How many neurons per layer?** (width)
3. **What activation functions?**
4. **How are layers connected?** (architecture)

| Architecture | Use Case | Key Feature |
|-------------|----------|-------------|
| MLP (fully connected) | Tabular data | Every neuron connects to every neuron |
| CNN (convolutional) | Images | Learns spatial filters |
| RNN / LSTM | Sequences (older approach) | Carries state over time |
| **Transformer** | **Language, everything** | **Attention mechanism** |

<Callout type="idea">
The Transformer architecture (2017) is what powers all modern LLMs — GPT, Claude, Gemini, Llama. We'll connect the dots in Chapter 7.
</Callout>

## Parameter Count

A simple network: 3 inputs → 4 hidden → 1 output

- Layer 1: $3 \times 4$ weights + $4$ biases = **16 parameters**
- Layer 2: $4 \times 1$ weights + $1$ bias = **5 parameters**
- **Total: 21 parameters**

That's it. Just 21 numbers to tune. But modern networks...

| Model | Parameters |
|-------|-----------|
| Simple MLP | 21 |
| LeNet (1998) | 60,000 |
| ResNet-50 (2015) | 25 million |
| GPT-2 (2019) | 1.5 billion |
| Modern open-weight LLMs | 7 billion to 70+ billion |
| Frontier LLMs | Tens to hundreds of billions+ (architecture-dependent) |

Same idea — just more parameters, more data, and more compute.
