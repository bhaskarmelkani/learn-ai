# Connecting to LLMs

Everything we've learned builds directly to Large Language Models.

## The Core Idea: Next Token Prediction

An LLM is a neural network trained on one task:

> Given a sequence of tokens, predict the **next token**.

Input: `"The cat sat on the"`
Output: probability distribution over all possible next tokens.

| Next Token | Probability |
|-----------|------------|
| mat | 0.25 |
| floor | 0.18 |
| roof | 0.12 |
| table | 0.08 |
| dog | 0.02 |
| ... | ... |

That is the core pretraining task. A base model learns broad language patterns from next-token prediction, and then chat assistants are usually further shaped with instruction tuning and preference training so they behave helpfully in conversation.

## Tokens, Not Words

A token is not always a whole word. Depending on the tokenizer, a token might be:

- A whole word like `cat`
- Part of a word like `ing`
- Punctuation like `.`
- Whitespace or formatting

This matters because LLMs do not literally think in words. They operate on token sequences, which is why code, punctuation, and spacing patterns matter during generation.

## Why Chat Models Feel Helpful

If next-token prediction is the core task, why does a chatbot answer questions, follow instructions, and write code?

Because the model has seen huge numbers of language patterns where a prompt is followed by a useful continuation. After pretraining, additional tuning teaches the model to continue text in the style of a helpful assistant.

| Stage | What it teaches |
|-------|-----------------|
| Pretraining | General language, facts, patterns, code structure |
| Instruction tuning | Follow requests in a chat-like format |
| Preference / safety tuning | Be more useful, safer, and better aligned with human expectations |

## What Makes LLMs Special?

### 1. The Transformer Architecture

The key innovation (2017 paper: "Attention Is All You Need"):

**Self-attention** — every token can look at every other token in the sequence to decide what matters.

```
"The cat sat on the [???]"

 The → (low attention)
 cat → (medium attention)
 sat → (high attention — verb determines what comes next)
 on  → (high attention — preposition constrains the answer)
 the → (medium attention)
```

This lets the model capture long-range relationships in text.

### 2. Scale

| Component | Typical LLM |
|-----------|------------|
| Parameters | Billions to hundreds of billions+ |
| Training data | Billions to trillions of tokens |
| Training cost | Very large, often millions of dollars or more |
| Vocabulary | ~100,000 tokens |
| Context window | Thousands to hundreds of thousands of tokens |

### 3. Emergent Abilities

As models get larger and training improves, some capabilities become much more reliable even though they were not programmed as explicit hand-written rules:
- Reasoning and chain-of-thought
- Code generation
- Translation between languages
- Following complex instructions
- In-context learning (learning from examples in the prompt)

## How Text Generation Works

```
Step 1: Input  → "The cat"
        Output → "sat"  (sampled from probability distribution)

Step 2: Input  → "The cat sat"
        Output → "on"

Step 3: Input  → "The cat sat on"
        Output → "the"

Step 4: Input  → "The cat sat on the"
        Output → "mat"

Result: "The cat sat on the mat"
```

Each step is one forward pass through the entire neural network. The model generates text one token at a time, feeding its output back as input.

<Callout type="analogy">
LLM text generation is like autocomplete on your phone — but trained on the entire internet, with billions of parameters, running hundreds of times per response.
</Callout>

## The Mental Model Progression

$$\text{Linear} \xrightarrow{\text{add layers}} \text{Neural Net} \xrightarrow{\text{add attention}} \text{Transformer} \xrightarrow{\text{scale up}} \text{LLM}$$

Each step is an incremental addition, not a revolution:

1. **Linear regression**: $y = wx + b$ → learns straight lines
2. **Add non-linearity**: stack layers with activations → learns complex functions
3. **Add attention**: tokens look at each other → learns language structure
4. **Scale**: billions of parameters + vast data → broader and more reliable capabilities

<Checkpoint prompt="If an LLM is 'just next-token prediction', why can it answer questions?">
Because answering a question is also a text continuation pattern. The model has learned that after a question, some continuations are much more likely to look like useful answers than others. Chat tuning then reinforces those helpful continuations even more.
</Checkpoint>

<Callout type="idea" title="The Big Picture">
There is no magic. LLMs are functions with parameters, trained with gradient descent, on the task of next-token prediction. The same $w \leftarrow w - \alpha \frac{\partial L}{\partial w}$ from Chapter 2 trains every model from linear regression to Claude.

The only differences are **scale** (more parameters), **data** (more examples), and **architecture** (Transformers with attention). The fundamental algorithm hasn't changed.
</Callout>
