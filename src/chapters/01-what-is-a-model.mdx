Imagine three products:

- a weather app that predicts tomorrow's rain chance
- a spam filter that decides whether an email is junk
- a house-pricing tool that estimates a sale price

All three do the same high-level thing: they take some **input** and produce some **output**.

That is the core idea of a model.

$$y = f(x)$$

A model is a function that maps inputs to outputs.

That sounds small, but it is the foundation of everything from the simplest regression model to a large language model.

## Everyday Examples

| Input $x$ | Output $y$ | Model |
|-----------|-----------|-------|
| House features (size, rooms, location) | Price | House-pricing model |
| Email text | Spam or Not Spam | Spam filter |
| Photo pixels | "Cat" or "Dog" | Image classifier |
| Sequence of tokens | Next token | Language model |

## Why "Model"?

A model does **not** recreate the whole world. It captures enough of a pattern to be useful.

> A model is a simplified pattern-finder that helps us make a prediction for a new example.

## The Two Key Questions

Every model is defined by two questions:

1. **What is the function's shape?** (architecture)
2. **What are the function's numbers?** (parameters)

Those two questions stay with us all the way through the course.

<Callout type="idea">
If you understand that a model is a function with learned numbers, you already have the basic mental model for all of AI.
</Callout>

## What Comes Next

There are two broad kinds of problems we keep returning to:

1. predict a **number**
2. make a **decision** between categories

We start with number prediction because it gives the clearest first picture of how models learn.

<Exercise title="Mental Habit">
When you look at any AI product, ask two questions before anything else: **what goes in and what comes out?** and **which numbers inside the model are being learned?**

If learners leave with only one instinct, it should be that one.
</Exercise>
