# Classification

So far we've predicted **numbers** (prices, temperatures). This is **regression**.

But many real problems ask: **which category?**

| Task | Input | Output |
|------|-------|--------|
| Spam detection | Email text | Spam / Not Spam |
| Medical diagnosis | Patient data | Disease A / B / C |
| Image recognition | Pixels | Cat / Dog / Bird |
| Sentiment analysis | Review text | Positive / Negative |

## Regression vs Classification

| | Regression | Classification |
|--|-----------|---------------|
| **Output** | Continuous number | Discrete category |
| **Example** | 350,000 | "Spam" |
| **Loss** | MSE (mean squared error) | Cross-entropy |

## The Sigmoid Function

To turn a number into a probability (between 0 and 1), we use the **sigmoid function**:

$$\sigma(z) = \frac{1}{1 + e^{-z}}$$

Where $z = wx + b$ (our familiar linear model).

Properties:
- $\sigma(0) = 0.5$
- Large positive $z$ → output near 1
- Large negative $z$ → output near 0

```
  1.0 │           ·········
      │         ·
      │        ·
  0.5 │·······●·······
      │      ·
      │     ·
  0.0 │·····
      └──────────────────── z
```

## Try It: Classify Points

Adjust w and b to separate the red (class 0) and blue (class 1) points. The dashed line at 0.5 is the decision boundary.

<Exercise title="Boundary Intuition">
Try changing only one parameter at a time:

- Change **b** while keeping **w** fixed. Watch the boundary slide left and right.
- Change **w** while keeping **b** fixed. Watch how sharply the probabilities switch from 0 to 1.

This is the classification version of the same intuition from linear regression: one number controls the baseline, another controls sensitivity.
</Exercise>

This gives us **logistic regression** — a classification model:

$$P(\text{spam}) = \sigma(wx + b)$$

If $P > 0.5$, predict "spam". Otherwise, "not spam".

<Callout type="idea">
The sigmoid squashes any number into a probability. This is the building block for classification — and it shows up again inside neural networks as an "activation function."
</Callout>

## Multi-Class: Softmax

For more than two categories, sigmoid generalizes to **softmax**:

$$P(y = k) = \frac{e^{z_k}}{\sum_{j} e^{z_j}}$$

This converts a vector of raw scores into probabilities that sum to 1.

> Softmax is how LLMs produce probability distributions over the entire vocabulary for the next token.

<Checkpoint prompt="Why not just threshold the raw score wx + b directly?">
Because classification is easier to reason about when the output behaves like a **probability**. The sigmoid maps any raw score into the 0 to 1 range, which makes both the interpretation and the loss function much cleaner.
</Checkpoint>
