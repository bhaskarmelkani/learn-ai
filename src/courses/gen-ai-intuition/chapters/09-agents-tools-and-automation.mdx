An **agentic workflow** usually means a system that can do more than answer once.

It can:

- plan steps
- use tools
- retrieve information
- call other systems
- revise its own output
- keep working toward a goal across multiple actions

That sounds powerful, but the practical beginner question is:

> Does this task really need an agent, or would a simpler workflow do better?

## Three Levels of Workflow Complexity

| Pattern | What it does | When it fits |
|-----------|-------|-------|
| Single-turn chat | one answer from one prompt | drafting, explanation, brainstorming |
| Structured workflow | prompt plus retrieval, schema, or review step | summaries, extraction, document Q&A |
| Agentic workflow | multi-step plan with tool use and iteration | research, coordination, multi-system tasks |

Many teams overuse the word **agent** when they really mean **workflow**.

That matters because complexity adds new failure modes:

- the wrong tool is used
- the plan drifts
- the system keeps going when it should stop
- the user loses visibility into where the answer came from
