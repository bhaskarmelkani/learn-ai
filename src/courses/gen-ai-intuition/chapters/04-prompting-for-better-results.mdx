The best prompt upgrades are usually not fancy.

They are practical:

- make the task narrower
- show one example
- define the format
- set boundaries
- ask the model to reveal uncertainty

## Four Reliable Upgrades

### 1. Add examples when the task is easy to misunderstand

If you want a particular style or answer shape, one example often teaches faster than another paragraph of explanation.

### 2. Add constraints when "good" is too vague

Useful constraints include:

- word limits
- required sections
- things to avoid
- what to do when the evidence is weak

### 3. Ask for structure when the output needs review

A structured answer is easier to scan, compare, verify, or reuse in another step.

### 4. Iterate like a collaborator, not a gambler

Good prompting often looks like:

1. try a first version
2. inspect what was weak
3. tighten the task or format
4. run again

<PromptComparison />

The design ideas above become more vivid when you run them on a weak local model.

When the model is tiny, vague prompts break faster and structured prompts help more obviously.
