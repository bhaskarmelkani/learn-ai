The model cannot read your intentions directly.

It only sees the text you gave it and the context that came before.

That means a prompt works best when it tells the model:

- what job to do
- who the output is for
- what context matters
- what shape the answer should take

## A Prompt Is an Interface

Beginners often hear "prompt engineering" and imagine hidden magic words.

A better frame is simpler:

> Prompting is interface design for a probabilistic system.

If the system is ambiguous, your prompt needs to reduce ambiguity.

If the system needs a specific answer format, your prompt should specify the format.

If the system needs to know what matters most, your prompt should rank the priorities.

## A Strong First Template

When in doubt, use this sequence:

1. state the task
2. give the necessary context
3. name the audience
4. define the output shape
5. tell it what uncertainty to admit

For example:

| Weak prompt | Better first prompt |
|-----------|-------|
| "Summarize this." | "Summarize this for a busy teammate in 4 bullets: main point, decision, blocker, and next step. If something is unclear, say so." |
