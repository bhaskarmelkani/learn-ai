When you type a prompt, a language model does not receive meaning directly.

It receives a sequence of **tokens**: chunks of text that get turned into numbers.

Then it does a repeated job:

1. read the tokens in context
2. estimate which token should come next
3. choose one
4. repeat until the answer ends

That is why the clearest beginner mental model is:

> An LLM is a next-token prediction engine wrapped in a useful chat interface.

## Step 1: Text Becomes Tokens

Words are not always tokens one-to-one. A token might be:

- a whole word
- part of a word
- punctuation
- a code symbol

That matters because tokenization affects:

- cost
- context length
- how the model chunks meaning

The same idea shows up again when you think about context windows.

It is not just **your** words that count.

System instructions, examples, chat wrappers, and the output you still want all spend the same finite budget.

<ContextBudgetLab />

## Step 2: Context Shapes the Next Step

A model does not generate all at once. At each step it looks at the current context window and predicts what token is likely next.

That is why changing earlier instructions can change later output. The model is not writing from an internal outline you can fully inspect. It is continuously recomputing what should come next from the context it can currently see.

<TokenPredictionDemo />

## Step 3: Sampling Changes the Feel

Two settings often matter to beginners:

- **temperature**: how sharp or spread out the next-token choice is
- **max output length**: how long the model is allowed to keep going

Low temperature usually feels more stable and repetitive. Higher temperature often feels more varied and creative, but also less controlled.

<Callout type="tip" title="Useful simplification">
  You do not need transformer math to use Gen AI well. The durable intuition is
  that the model sees tokens, uses context, and keeps predicting the next step.
</Callout>
