This chapter is intentionally **dated**.

The snapshot below is framed for **March 2026**. Specific products, plans, and model names will keep changing.

The durable part is the selection logic:

- what job are you trying to do?
- what context does the tool need?
- how much structure, retrieval, or collaboration do you need?
- what human review is still required?

## Category Map

| Category | Good for | Representative tools as of March 2026 |
|-----------|-------|-------|
| General chat assistants | drafting, brainstorming, explanation, everyday work | ChatGPT, Claude, Gemini |
| Research and source-heavy tools | web-backed answers, synthesis with sources | Perplexity, ChatGPT Deep Research, Microsoft 365 Copilot Researcher |
| Notebook and source-grounded study tools | asking questions over provided source material | NotebookLM, project-based chat workspaces in ChatGPT and Claude |
| Coding assistants | code generation, explanation, refactors, IDE help | GitHub Copilot, Cursor, Claude in coding environments, OpenAI Codex-style agents |
| Multimodal creation tools | image, audio, video, and design drafts | Midjourney, Runway, Adobe Firefly, ElevenLabs |
| Workflow and automation tools | connecting prompts to repeatable steps and business systems | Microsoft Copilot agents, Zapier AI-style workflows, no-code automation stacks |

<Callout type="tip" title="How to read this snapshot">
Do not ask "Which tool is best?" first. Ask "Which category fits the job, and what evidence would tell me it is working?"
</Callout>

## Practical Selection Criteria

When comparing tools in the same category, look for:

1. how well the tool handles your actual task
2. whether it can use the right sources or documents
3. how easy it is to review, export, and reuse the output
4. what privacy or admin controls matter in your environment
5. whether the workflow is individual, team-based, or integrated into another system
