At large scale, a transformer trained on next-token prediction is already a **language model**.

That base model learns broad statistical patterns of language, code, and structure from its training data.

## Base Model vs Assistant

A base language model is optimized to continue text.

An assistant-style LLM is what you get when extra steps are added on top, such as:

- **instruction tuning** so the model follows tasks more reliably
- **preference optimization / alignment tuning** so outputs better match human preferences
- product-level systems such as retrieval, tools, memory, moderation, and UX constraints

That means it is more accurate to say:

1. pretraining creates a base language model
2. alignment steps can make that model more useful as an assistant
3. product systems around the model shape the real user experience

## Important Failure Modes

Even a strong LLM product can still fail in familiar ways:

- hallucinating unsupported details
- following the wrong instruction from messy context
- sounding confident when evidence is weak
- failing silently on long or noisy context
- inheriting bias from training data or retrieval data

The right mental model is not "the LLM knows" but "the system predicts likely continuations under learned patterns and product constraints."

## Retrieval and Tools Are System Layers

Modern AI products often add **retrieval**, **tool use**, or **workflow logic** around the model.

That can make the product much more useful, but it also means the visible behavior is not coming from the base model alone.
