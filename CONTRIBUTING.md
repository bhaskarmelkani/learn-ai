# Contributing

This repository expects substantive code and content contributions to be made through coding agents that follow the repo instructions and mirrored skill docs.

## Contribution Policy

- Use a coding agent such as Codex, Claude Code, or a similar tool that can follow repo-local instructions.
- Start from [AGENTS.md](/Users/bhaskar.melkani/Documents/Projects/bhaskar/learn-ai/AGENTS.md) or [CLAUDE.md](/Users/bhaskar.melkani/Documents/Projects/bhaskar/learn-ai/CLAUDE.md).
- Use the public workflow mirrors in [docs/skills/README.md](/Users/bhaskar.melkani/Documents/Projects/bhaskar/learn-ai/docs/skills/README.md).
- Run `pnpm lint`, `pnpm test`, `pnpm build`, and `pnpm e2e` when relevant before opening a PR.

This policy exists because the repo encodes contributor judgment in reusable skills and playbooks. That keeps curriculum structure, browser-AI labs, design-system usage, and review standards consistent.

## Recommended Workflow

1. Read [docs/contributing-courses.md](/Users/bhaskar.melkani/Documents/Projects/bhaskar/learn-ai/docs/contributing-courses.md).
2. Choose the right skill flow from [docs/skills/README.md](/Users/bhaskar.melkani/Documents/Projects/bhaskar/learn-ai/docs/skills/README.md).
3. Make incremental changes that follow existing patterns.
4. Run the required validation commands.
5. Include screenshots or notes for visible UI changes.

Small typo reports or issue filing do not require this workflow, but proposed code, curriculum, or interactive-lab changes do.
